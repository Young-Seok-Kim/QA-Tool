Main Dialog가 열림
-> Thread First가 실행됨
-> VIEW 버튼을 클릭
-> Thread First가 에서 캠을 이용해 이미지 하나를 받아옴->Thread Second에서 캠을 이용해 이미지 하나를 뿌려줌 -> .... 무한반복(이미지를 받고 뿌리는 것이 무한반복 되면 영상이된다)
-> 이미지를 캡쳐 하고 Compare 버튼을 누르면 이미지의 유사도가 나옴
-> VIEW Dialog를 닫으면 Thread Second가 끝나면서 Thread First 재실행

설정값을 지정하는 Dialog도 하나 만들어야될듯? -> 'VIEW Dialog에서 캡쳐한 n번째 이미지를 n초후에 비교한다.'와 같은 기능을 구현하기 위함
Thread First의 역할 : Cam에서 이미지를 읽어온다.
Thread Second의 역할 : Thread First에서 읽어온 이미지를 VIEW Dialog에 그려준다.
